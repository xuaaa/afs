第十七天笔记

DOM

1.DOM--->Document Object Model

2.DOM定义了表示和修改文档所需要的方法。DOM对象即为宿主对象，由浏览器厂商定义，用来操作 html和xml功能的一类对象集合，也有人称DOM是对html以及xml的标准编程接口。

DOM基本操作

1.d对节点的增删查看。

查看元素节点

  1. 1.document:代表整个文档。
  2. 2.document.getElementById()//元素id在ie8以下的浏览器，不区分id大小写，匹配name属性的元素。
  3. 3.document.getElementsByTagName()//标签名
  4. 4.document.getElementsByName()//需注意只有部分标签name可生效。（表单，表单元素，img，ifame）
  5. 5.document.getElementsByClassName()//类名-->ie8和ie8以下的ie版本没有
  6. 6.document.querySelector()//css选择器。ie6和ie7没有，不是实时。
  7. 7.document.querySelectorAll()//css选择器。ie6和ie7没有，不是实时。

遍历节点树((都为属性) 兼容性好 但是会选出来文本或者注释)：

  1. 1.ParentName-->父节点（最顶端的ParentName为#document）
  2. 2.ChildNodes-->子节点们。
  3. 3.fristChild-->第一个子节点
  4. 4.lastChild-->最后一个子节点
  5. 5.nextSibling/PreviousSibling-->后一个兄弟节点/前一个兄弟节点

基于元素节点树的遍历((都为属性) 不兼容都为IE9及以下不兼容)：

  1. 1.ParentElement-->返回当前元素的父元素的节点（ie不兼容）
  2. 2.Children-->只返回当前元素的元素子节点
  3. 3.node.childElementCount===node.children.length --->当前元素节点的子元素节点的个数
  4. 4.FristElementChild--->返回第一个元素子节点
  5. 5.LastElementChild--->返回最后一个子元素节点
  6. 6.nextElementSibling/PreviousElementSibling--->返回后一个/前一个兄弟子元素节点

节点的四个属性：

  1. 1.nodeName-->元素标签名，以大写形式表示，只读
  2. 2.nodeValue--->text节点或Comment节点的文本内容。可读写，不可更改。
  3. 3.nodeType--->只读
  4. 4.attributes--->属性节点的集合
  5. 节点的一个方法：Node.hasChildNodes();

DOM结构树

查找元素：

  1. 1.getElementById() 定义在Document.prototype 上，即Element节点不能访问
  2. 2.getElementsByName() 定义在HTMLDocument.prototype上，即非html元素不能使用（例：xml,element）
  3. 3.getElementsByTagName() 定义在Document.prototype上，和Element.prototype 兼容性最好
  4. 4.HTMLDocument.prototype上定义了常用属性（例如：body,head）分别指向文档中的和标签
  5. 5.Document.prototype定义了documentElement属性，指代文档根元素即 标签
  6. 6.getElementsByClassName 、querySelectorALl 、querySelector 在Document.prototype 和 Element.prototype 中都有定义。
